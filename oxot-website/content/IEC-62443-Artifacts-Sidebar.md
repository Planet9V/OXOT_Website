### STRATEGIC ARTIFACTS (2)

#### 1. Business Rationale for CSMS
**IEC 62443 Reference:** 2-1 (Section 4.2.2)
**Purpose:** Document financial, HSE, and operational impact justification for cybersecurity program
**Who Needs This:** Executive leadership, board of directors, CFO
**What It Contains:** Risk quantification, regulatory obligations, business continuity requirements
**Why It Matters:** Secures executive buy-in and budget allocation for CSMS implementation

#### 2. Cybersecurity Requirements Specification (CRS)
**IEC 62443 Reference:** 3-2 (Section 5.3)
**Purpose:** Translate risk assessment findings into technical procurement requirements
**Who Needs This:** Procurement, system integrators, vendor management
**What It Contains:** Security Level targets (SL-T), zone/conduit definitions, compensating controls

#### 3. High-Level Risk Assessment Template
**IEC 62443 Reference:** 3-2 (Initial Risk Assessment)
**Purpose:** Rapidly identify highest-consequence zones through structured consequence analysis
**Who Needs This:** Asset owners, operations managers, safety engineers
**What It Contains:** System Under Consideration (SUC) definition, zone partitioning, initial SL-T assignment
**Why It Matters:** Focuses resources on protecting the most critical assets first

**IEC 62443 Reference:** 3-2 (Detailed Risk Assessment)
**Purpose:** Comprehensive threat modeling, vulnerability analysis, and residual risk calculation
**Who Needs This:** Security analysts, control engineers, system integrators
**What It Contains:** Threat vectors, likelihood/impact matrices, countermeasure specifications, residual risk
**Why It Matters:** Provides evidence-based justification for security investments and control selection

**Who Needs This:** Network architects, system integrators, auditors
**What It Contains:** Network diagrams (logical/physical), zone definitions, conduit security controls, data flow maps

### OPERATIONAL ARTIFACTS (4)

#### 6. Incident Response Plan (IRP) for OT/ICS
**IEC 62443 Reference:** 2-1 (Section 3.3.4.1)
**Purpose:** Define procedures for detecting, containing, and recovering from OT cyber incidents
**Who Needs This:** Security operations center (SOC), plant operators, incident response team
**What It Contains:** Incident classification, response procedures, escalation paths, communication protocols
**Why It Matters:** Reduces incident response time by 60-80% and prevents cascading failures
**[Download Template →]** *(Gated Content)*

#### 7. Patch Management Procedure (OT-Safe)
**IEC 62443 Reference:** 2-1 (Section 3.4.2.1) + 2-3 (Technical Report)
**Purpose:** Manage vulnerability remediation without causing operational disruption
**Who Needs This:** IT/OT operations, maintenance teams, change advisory board
**What It Contains:** Vulnerability monitoring, risk-based prioritization, testing procedures, deployment windows, rollback plans
**Why It Matters:** Addresses the #1 OT security challenge—balancing patching urgency with availability requirements
**[Download Template →]** *(Gated Content)*

#### 8. Cyber Factory Acceptance Test (CFAT) Plan
**IEC 62443 Reference:** 3-3 (System Testing)
**Purpose:** Verify security controls before system shipment from integrator facility
**Who Needs This:** System integrators, commissioning engineers, quality assurance
**What It Contains:** Security test procedures, acceptance criteria, configuration verification checklists
**Why It Matters:** Catches 70% of security configuration errors before expensive on-site deployment
**[Download Template →]** *(Gated Content)*

#### 9. Cyber Site Acceptance Test (CSAT) Plan
**IEC 62443 Reference:** 3-3 (System Validation)
**Purpose:** Validate achieved Security Level (SL-A) after full system integration at operational site
**Who Needs This:** Asset owners, commissioning teams, auditors
**What It Contains:** Zone boundary testing, authentication verification, incident detection validation, SL-A evidence
**Why It Matters:** Establishes verified security baseline for operational phase and proves SL-A ≥ SL-T
**[Download Template →]** *(Gated Content)*

### PROCUREMENT ARTIFACTS (3)

#### 10. IEC 62443 RFP Template (Product Suppliers)
**IEC 62443 Reference:** 4-1 (SDL) + 4-2 (Component Requirements)
**Purpose:** Procurement language requiring ISASecure certification and verifiable SL-C claims
**Who Needs This:** Procurement officers, vendor management, legal
**What It Contains:** Required SL-C levels, ISASecure SDLA/CSA certification requirements, PSIRT obligations, support SLAs
**Why It Matters:** Transfers 50-70% of security burden to suppliers through enforceable contracts

#### 11. System Integrator Qualification Checklist
**IEC 62443 Reference:** 2-4 (Service Provider Requirements)
**Purpose:** Evaluate system integrators against IEC 62443-2-4 security capabilities
**Who Needs This:** Procurement, project managers, asset owners
**What It Contains:** 12-category assessment (security design, config mgmt, remote access, patch mgmt, personnel qualifications)
**Why It Matters:** Prevents "secure components, insecure system" failure mode by qualifying integrators

**IEC 62443 Reference:** 2-1 (Section 3.3.3.2)
**Purpose:** Third-party risk evaluation for service providers and technology vendors
**Who Needs This:** Vendor risk management, procurement, legal
**What It Contains:** Security program maturity, incident history, vulnerability disclosure policy, insurance, contractual protections
**Why It Matters:** Addresses supply chain risk (45% of energy sector breaches) through structured vendor vetting
